# SySeBa 2.0.0 - C nativo

[English](https://github.com/okno/SySeBa/blob/main/docs/releases/v2.0.0.md) |
[Indice documentazione](https://github.com/okno/SySeBa/blob/main/docs/README.it.md)

SySeBa 2 è una riscrittura completa C11 del servizio di backup continuo.
Mantiene configurazione e semantica restore esistenti eliminando la dipendenza
runtime da Python.

## Asset della release

- `syseba_2.0.0_amd64.deb`: pacchetto Debian/Ubuntu x86_64
- `syseba-2.0.0-1.x86_64.rpm`: pacchetto Linux RPM x86_64
- `syseba-2.0.0-linux-x86_64.tar.gz`: bundle Linux portabile
- `SySeBa-2.0.0-windows-x86_64.zip`: bundle Windows 11/Server
- `SySeBa-2.0.0-windows-x86_64-setup.exe`: installer Windows
- `SySeBa-2.0.0-macos-universal.dmg`: immagine macOS Universal 2
- `syseba-2.0.0-source.tar.gz`: sorgente C e dipendenze vendorizzate
- `SHA256SUMS`: hash di integrità
- `release-manifest.txt`: provenienza build e stato verifiche

## Funzioni principali

- Watcher inotify e `ReadDirectoryChangesW`, con polling di ripiego
- Sincronizzazione iniziale parallela e code worker limitate
- Migrazione transazionale dei vecchi schemi SQLite
- Dashboard CLI bilingue e browser restore
- Web UI incorporata protetta da token sulla porta configurabile `8765`
- Integrazione systemd, Windows Service e LaunchDaemon macOS
- Validazione percorsi, replace atomico e lock istanza singola
- Migrazione a servizio fermo, snapshot, health check e rollback esatto

## Aggiornamento da Python

Usa un checkout separato per creare lo snapshot di `/opt/syseba`:

```bash
git clone https://github.com/okno/SySeBa.git /root/SySeBa-release
cd /root/SySeBa-release
sudo ./scripts/migrate-from-python.sh
```

Lo script conserva configurazione, database, log, token Web e stato servizio.
Il rollback ripristina la versione esatta salvata.

## Verifica

La pipeline esegue CTest Linux, integrazione Windows, integrità pacchetti,
analisi statica, sanitizer e controlli Web UI desktop/mobile.

Il binario macOS è cross-compilato e verificato strutturalmente, ma non
eseguito su hardware Apple. Gli artefatti Windows/macOS non sono firmati e il
DMG non è notarizzato. Verifica `SHA256SUMS` prima dell'installazione.

Il workflow GHCR ha scaricato nuovamente la Release, verificato nomi e SHA-256,
pubblicato un package pubblico collegato alla repository e completato un pull
anonimo con estrazione.
